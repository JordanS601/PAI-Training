Step-by-Step Reference Guide

Use this written guide alongside or after the video. Each section includes a platform screenshot for reference.

Section 1: Access the Document Collector Dashboard

Sign in to Law Practice AI and navigate to the Document Collector module from the main portal. The dashboard displays all active and past document collection requests in a list view.

Key elements on the dashboard:

  • Title -- the client's name tied to each request
  • Date Requested -- when the collection was created
  • Documents Uploaded -- progress tracker (e.g., "2/9" means 2 of 9 documents received)
  • Status -- either "Created" (no uploads yet) or "Pending Upload" (partial uploads)
  • Progress Bar -- visual indicator of completion

Your account has a document usage limit shown at the top left (e.g., "170 documents left, 30 of 200 used"). Monitor this and notify your administrator if you are approaching the cap.

Use the Search bar to find specific clients and the "Export to Excel" button to download reports.

Document Collector Dashboard -- showing active collections, status, and document progress
Tip: Use your firm's naming convention consistently when creating requests. This makes it much easier to search and track collections as your list grows.

Section 2: Create Outbound Email and SMS Templates

Before sending your first document request, set up your Outbound Templates. Navigate to "Outbound Templates" in the left sidebar.

There are three template types, each available as both Email and SMS:

  • Upload Request Email -- the initial message sent to the client with the upload link
  • Pending Document Reminder -- follow-up messages listing which documents are still missing
  • Invalid File Template -- notification sent when a client uploads an unsupported file type

To create or edit a template:

  • Select the plus (+) icon to add a new template
  • Choose between Text or HTML format (Text is recommended for simplicity)
  • Select the delivery type: Email or SMS
  • Enter your message content, then click "Save Template"
  • Repeat for each template type you need
Outbound Templates -- Upload Request Email shown in Text format with merge variables
Warning: Never delete or alter merge variables like {client_name}, {organization_name}, or {requested_documents}. Broken variables will display raw code to the client instead of their actual information.

Section 3: Build the Reminder Schedule

Navigate to "Reminder Schedule" in the left sidebar. This is where you configure the automated follow-up sequence that runs after a document request is sent.

Before adding reminder steps, configure the Contact Window on the right side of the screen:

  • Start Time / End Time: Set to your firm's business hours (recommended: 9:00 AM to 5:00 PM)
  • Timezone: Select your firm's local timezone
  • Weekday: Select "Mon - Fri" (recommended). This ensures clients only receive reminders during business hours on weekdays.
Important: Confirm the reminder is set to send to the client, not the provider. The contact window should match your firm's office hours so clients receive messages at appropriate times.

To build the reminder sequence, click the plus (+) button to add your first step. Each step in the sequence needs:

  • Reminder type: Email or SMS
  • Template: Select from the templates you created in Section 2
  • Waiting period: How long to wait before sending this step (e.g., 1 day, 7 hours for same-day)
Reminder Schedule -- automated sequence with 1-day intervals and Mon-Fri contact window

Section 4: Configure Reminders and Follow-Up Timing

Continue adding steps to your reminder sequence. A typical sequence looks like this:

  • Step 1: Email -- Upload Request (sent immediately after document request is created)
  • Step 2: Wait 1 day, then Email -- Pending Document Reminder
  • Step 3: Wait 1 day, then Email -- Pending Document Reminder
  • Step 4: Wait 1 day, then SMS -- Pending Document Reminder
  • Step 5: Wait 1 day, then Email -- Final Pending Reminder
  • Final Step: Add an Invalid File Template step to handle cases where a client uploads an unsupported file format

Each step must have a waiting period between it and the previous step. Without a waiting period, messages will send simultaneously, which will overwhelm the client.

Best practice: Start with a simple sequence (3 to 5 steps). You can always add more steps later once you see how clients respond. Mix email and SMS for better response rates.

Section 5: Invalid File Handling

The Invalid File Template is triggered automatically when a client uploads a file type that the system cannot process. This notification lets the client know the issue and asks them to re-upload in an accepted format.

Make sure your invalid file template:

  • Clearly explains what went wrong (the file type was not supported)
  • Lists accepted file formats if possible
  • Provides a link or instructions for re-uploading
  • Maintains a professional and helpful tone
Invalid File Template -- SMS version notifying client of unsupported file type

After configuring all steps and the invalid file notification, click "Save Sequence" at the top of the Reminder Schedule page. Review the entire sequence before saving to verify each step has the correct template and waiting period assigned.

Section 6: Send the Document Request to the Client

With templates and reminders configured, you are ready to send your first document request. Click "Request New Documents" in the left sidebar.

Fill in the client details:

  • Client Full Name -- exactly as it appears in the case file
  • Client Email -- double-check for typos; an incorrect email means the client never receives the request
  • Client Phone Number -- used for SMS reminders

Then select the document types needed:

  • Documents: Medical Record, Police Report, Medical Bill, Repair Order
  • Images: Property Damage Evidence, Injury Evidence, Incident Location
  • Other Documents: Use this field for custom types not in the standard list (e.g., "Employer's Incident Report")

Confirm all details are correct, then click "Request Documents" to submit. The client will immediately receive the first message in your outbound sequence.

Request New Documents form -- client details, document types, and image types selection
Best practice: Request more document types rather than fewer. It is easier to discard an unnecessary document than to send a second request for something you forgot.

Section 7: Requesting Documents from Providers

In addition to collecting documents directly from clients, you will also need to request records from medical providers, insurance companies, and other third parties. This process requires a HIPAA Authorization Form signed by the firm's client before any records can be released.

Required: No provider will release medical records without a valid HIPAA authorization on file. This form must be signed by the client before you initiate any provider requests. Confirm with your supervising attorney or office manager that the signed authorization is in the case file before proceeding.

Provider Request Workflow:

  • Step 1: Confirm a signed HIPAA Authorization Form is on file for the client
  • Step 2: Submit the document request through the Document Collector using the same process as client-facing requests
  • Step 3: The provider receives the request and processes it. Some providers may require a secondary consent form specific to their organization
  • Step 4: If secondary consent is requested, upload the completed form to the system
  • Step 5: Once authorization is confirmed, documents are typically received within 48 hours
Note: The provider-facing workflow within the Document Collector platform has not yet been fully documented. This section will be updated with platform-specific screenshots and video walkthroughs once the process documentation is provided by the Law Practice AI team. The steps above reflect the standard operating procedure.

Cautionary Notes

  • Always include waiting periods between reminder steps. Without them, all messages fire at once and overwhelm the client.
  • Verify your templates before saving the sequence. Sending a blank or broken template to a client is not recoverable.
  • Match the contact window to your firm's hours. Clients receiving messages at 2 AM reflects poorly on the firm.
  • Direct reminders to the client, not the provider. The reminder sequence is designed for client-facing communication. Provider requests follow a separate workflow.
  • Review the full sequence before saving. Check that each step references the correct template and has the right delivery type (Email vs. SMS).

Tips for Efficiency

  • Use your firm's naming convention for every request. Consistent naming makes searching and reporting significantly easier.
  • Start with a simple reminder sequence (3 to 5 steps). You can always expand it later based on client response patterns.
  • Reuse templates across cases. Once your templates are set up, they apply to every new document request automatically.
  • Keep office hours as the default contact window. Only change this if your firm specifically operates on a different schedule.
  • Double-check client email and phone number before submitting. An incorrect email means the client never receives the request, and you will not know until days later when nothing comes back.

Section 8: Best Practices for PI Cases

  • Send document requests within 24 hours of client sign-up
  • Tell clients during intake to expect the automated email so they do not ignore it
  • If a client is unresponsive after 5 business days, make a personal phone call
  • If still no response after 10 business days, escalate to the supervising attorney
  • For photos: quality matters more than quantity -- a few clear, well-lit images are better than dozens of blurry ones
  • Medical records: be prepared to guide clients on how to request records from their provider, or flag for HIPAA authorization
